# 007. Registration FSM

**Статус:** todo  
**Фаза:** bot  
**Зависимости:** 006, 003, 004

## Описание

Реализовать пошаговую регистрацию нового пользователя: возраст → пол → с кем хочет общаться → язык. После завершения — переход в главное меню.

## Scope

- FSM states: `Age` → `Gender` → `Seeking` → `Language` → `Done`
- **Возраст:** числовой ввод, валидация 18–99, отказ при < 18
- **Пол:** inline-кнопки Парень / Девушка
- **Ищу:** inline-кнопки Парень / Девушка
- **Язык:** inline-кнопки RU / EN
- Сохранение в API: `POST /users/register`
- Повторный `/start` у зарегистрированного → главное меню
- FSM state в Redis (`fsm:{telegram_id}`)

## Acceptance criteria

- [ ] Новый пользователь проходит все 4 шага без ошибок
- [ ] Возраст < 18 → сообщение об отказе, регистрация не завершается
- [ ] Данные сохраняются в `users` + `profiles`
- [ ] Emit `user.registered` event
- [ ] Прерванная регистрация возобновляется с текущего шага
- [ ] Невалидный ввод (буквы в возраст) → повторный запрос

## Технические заметки

- Кнопки пола/ищу/языка — callback_query с prefix `reg:`
- После регистрации очистить FSM key
- `seeking` определяет match route (задача 010)
- Checkbox 18+ можно добавить как отдельный шаг перед возрастом (опционально)

## Out of scope

- OAuth / внешняя верификация возраста
- Редактирование анкеты (задача 024)
